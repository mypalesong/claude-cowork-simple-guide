MCP 연동 가이드 ​

Claude에게 새로운 능력을 추가하는 플러그인, MCP. 회사에서 쓰는 도구들과 Claude를 연결하면 업무가 훨씬 편해집니다.

MCP가 뭔가요? ​

**MCP(Model Context Protocol)**는 Claude가 외부 서비스와 대화할 수 있게 해주는 플러그인 시스템입니다.

비유하자면 이렇습니다:

  • Claude 자체 = 똑똑한 비서
  • MCP = 비서에게 주는 열쇠 묶음

열쇠가 없으면 비서가 아무리 똑똑해도 구글 드라이브 파일을 열어볼 수 없고, 슬랙 메시지를 검색할 수 없고, 캘린더를 확인할 수 없습니다. MCP는 이 열쇠를 Claude에게 건네주는 방법입니다.

핵심 개념

MCP를 설치하면 Claude가 직접 회사 도구에 접근합니다. "복사해서 붙여넣기" 할 필요 없이, "슬랙에서 찾아줘"라고 말하면 됩니다.

MCP로 할 수 있는 것들 ​

연동 서비스할 수 있는 것활용 예시
Google Drive파일 검색, 문서 읽기, 내용 요약"지난달 마케팅 보고서 찾아서 요약해줘"
Slack메시지 검색, 채널 요약, 스레드 정리"이번 주 #개발팀 채널 논의 정리해줘"
GitHub이슈 조회, PR 확인, 코드 리뷰 요약"이번 스프린트 열린 이슈 목록 보여줘"
Google Calendar일정 확인, 빈 시간 찾기, 미팅 잡기"다음 주 화~목 중 팀 미팅 가능한 시간 찾아줘"
사내 DB데이터 조회, 리포트 생성"이번 달 신규 고객 수 알려줘"
Notion페이지 검색, 위키 조회"온보딩 가이드 찾아서 핵심만 알려줘"

꿀팁

MCP는 계속 늘어나고 있습니다. 필요한 연동이 있다면 IT팀에 문의해보세요. 대부분의 업무 도구는 MCP로 연결할 수 있습니다.

MCP 설치 방법 ​

1단계: Claude Desktop 설정 열기 ​

Claude Desktop 좌측 하단의 설정(⚙️) 아이콘을 클릭합니다.

2단계: MCP 서버 추가 ​

설정 화면에서 "MCP Servers" 탭을 선택하고, "Add Server" 버튼을 클릭합니다.

3단계: 연동할 서비스 선택 ​

연결하고 싶은 서비스를 목록에서 선택합니다. 예를 들어, Google Drive를 연동하려면:

text
1. "Add Server" 클릭
2. "Google Drive" 선택
3. Google 계정 로그인 (회사 계정 사용)
4. 접근 권한 허용
5. 연결 완료!

4단계: 연결 확인 ​

대화창에서 아래처럼 물어보면 잘 연결됐는지 확인할 수 있습니다:

text
지금 연결된 MCP 서비스 목록 보여줘.

IT팀 확인 필요

회사에서 MCP를 사용하려면 IT팀의 승인이 필요할 수 있습니다. 먼저 IT팀에 "Claude Desktop MCP 사용 가능 여부"를 확인하세요. 회사 보안 정책에 따라 사용 가능한 MCP가 제한될 수 있습니다.

실전 활용 시나리오 ​

시나리오 1: 슬랙 논의 요약 ​

상황: 출장 다녀왔더니 슬랙 메시지가 500개 쌓여 있을 때

text
슬랙에서 이번 주 #마케팅팀 채널 논의 요약해줘.

알고 싶은 것:
- 주요 결정 사항
- 나한테 관련된 멘션이 있었는지
- 다음 주까지 해야 할 액션 아이템

시나리오 2: 구글 드라이브 파일 찾기 ​

상황: 파일 이름이 정확히 기억 안 나는데 지난달 보고서를 찾아야 할 때

text
구글 드라이브에서 지난달(2월)에 만든 보고서 파일 찾아줘.
"매출" 또는 "실적"이라는 단어가 들어간 문서로.
찾으면 핵심 내용 3줄로 요약해줘.

시나리오 3: 캘린더에서 빈 시간 찾기 ​

상황: 팀원 5명의 일정을 맞춰 미팅을 잡아야 할 때

text
내 캘린더에서 다음 주 빈 시간 찾아줘.

조건:
- 월~금, 오전 10시~오후 5시 사이
- 최소 1시간 블록
- 점심시간(12~1시) 제외
가능한 시간대를 목록으로 보여줘.

시나리오 4: GitHub 이슈 관리 ​

상황: 스프린트 회의 전에 이슈 현황을 빠르게 파악해야 할 때

text
GitHub에서 우리 프로젝트 이슈 현황 정리해줘.

보고 싶은 것:
- 이번 스프린트 열린 이슈 수
- 우선순위별 분류 (긴급/보통/낮음)
- 담당자별 할당 현황
표로 정리해줘.

주의사항 ​

권한 관리 ​

항목확인 사항
최소 권한 원칙필요한 권한만 허용하세요. "모든 파일 접근"보다 "특정 폴더만 접근"이 안전합니다
회사 계정 사용MCP 연동 시 반드시 회사 계정으로 로그인하세요. 개인 계정 사용은 보안 이슈가 됩니다
정기 점검더 이상 안 쓰는 MCP 연동은 해제하세요. 설정에서 연결된 서비스 목록을 주기적으로 확인합니다

보안 고려사항 ​

민감 정보 주의

  • 기밀 문서: MCP로 접근하는 문서에 기밀 정보가 포함되어 있을 수 있습니다. Claude에게 요약을 요청한 결과를 외부에 공유하지 마세요.
  • 개인정보: 고객 개인정보가 포함된 DB를 조회할 때는 회사 개인정보처리방침을 확인하세요.
  • 접근 로그: MCP를 통한 접근도 서비스에 로그가 남을 수 있습니다. 업무 목적으로만 사용하세요.

IT팀에 요청하기 ​

MCP 도입을 검토 중이라면, 아래 체크리스트를 IT팀과 함께 확인하세요.

MCP 도입 협의 체크리스트 ​

text
□ 사용 가능한 MCP 서버 목록 확인
□ 회사 보안 정책과의 호환성 검토
□ 직원별 접근 권한 범위 설정
□ 데이터가 외부로 전송되는 경로 확인
□ 개인정보 처리 관련 법적 검토 (PIPA 등)
□ MCP 연동 시 사용할 계정 정책 (개인/공용)
□ 문제 발생 시 지원 체계 (담당자, 연락처)
□ 도입 후 모니터링 및 감사 계획
□ 직원 교육 및 가이드 배포 일정
□ 시범 운영 범위 및 기간 결정

IT팀 요청 메일 예시 ​

text
제목: Claude Desktop MCP 연동 검토 요청

안녕하세요, [부서명] [이름]입니다.

업무 효율화를 위해 Claude Desktop의 MCP(Model Context Protocol) 기능
도입을 검토하고 있습니다.

연동 희망 서비스: Google Drive, Slack
예상 사용 인원: [N]명
주요 활용 목적: 문서 검색 및 요약, 채널 논의 정리

보안 정책 검토 및 테스트 계정 발급이 가능한지 확인 부탁드립니다.
감사합니다.

도입 전략

한 번에 모든 서비스를 연동하기보다, 가장 많이 쓰는 서비스 1~2개부터 시범 운영하는 것을 추천합니다. 효과를 확인한 후 점차 확대하면 IT팀 설득도 쉬워집니다.
