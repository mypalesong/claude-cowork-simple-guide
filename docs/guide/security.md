# 보안 & 컴플라이언스

> "이거 회사에서 써도 괜찮은 거야?" — 가장 많이 받는 질문에 대한 답을 정리했습니다.

AI 도구를 업무에 사용할 때 보안은 선택이 아니라 필수입니다. 이 페이지에서는 Claude Desktop Cowork를 **안전하게** 사용하는 방법을 실무자 눈높이에서 알려드립니다.

---

## 1. 데이터 처리 원칙

Claude가 내 데이터를 어떻게 다루는지 먼저 알아봅시다.

| 항목 | 내용 |
|------|------|
| **대화 내용 학습** | Enterprise 및 Team 플랜은 대화 내용을 모델 학습에 **사용하지 않습니다** |
| **데이터 저장** | 대화는 일정 기간 보관 후 삭제되며, 플랜별로 보관 정책이 다릅니다 |
| **파일 처리** | Cowork에서 읽은 파일은 대화 세션 내에서만 사용되며, 별도 저장되지 않습니다 |
| **전송 암호화** | 모든 데이터는 전송 중(TLS) 및 저장 시(AES-256) 암호화됩니다 |
| **제3자 공유** | 사용자 데이터를 제3자에게 판매하거나 공유하지 않습니다 |

::: tip 핵심 포인트
**회사에서 Team 또는 Enterprise 플랜을 사용 중이라면**, 여러분의 대화 내용이 AI 학습에 쓰이지 않습니다. IT팀에 현재 어떤 플랜인지 확인해보세요.
:::

## 3. 파일 접근 권한 관리

Cowork 모드에서는 Claude가 내 컴퓨터 파일에 접근합니다. **어떤 폴더를 열어줄지**가 보안의 핵심입니다.

### 허용해도 되는 폴더

```
✅ 현재 작업 중인 프로젝트 폴더
✅ 분석할 데이터가 있는 폴더 (민감정보 제거 후)
✅ 작성 중인 문서 폴더
```

### 절대 허용하면 안 되는 폴더

```
❌ C:\Users\[내 이름] 전체 (홈 폴더 루트)
❌ 사내 VPN/인증서 폴더
❌ .ssh, .aws, .env 등 인증 정보 폴더
❌ 비밀번호 관리 프로그램 데이터 폴더
❌ 인사/급여 관련 파일이 있는 폴더
❌ 고객 개인정보 DB 폴더
```

::: tip 실무 팁: 작업 폴더를 따로 만드세요
`D:\Claude작업` 같은 **전용 작업 폴더**를 만들고, 분석할 파일만 복사해서 넣은 뒤 그 폴더만 Cowork에 연결하면 안전합니다. 원본 파일은 건드리지 않아도 되고, 민감 정보를 실수로 노출할 위험도 줄어듭니다.
:::

## 4. 입력하면 안 되는 정보

아무리 편하더라도, 아래 정보는 **절대 Claude에 입력하지 마세요**.

### 🚫 절대 금지 목록

| 구분 | 구체적 예시 |
|------|------------|
| **개인 식별 정보** | 주민등록번호, 여권번호, 외국인등록번호 |
| **금융 정보** | 신용카드 번호, 계좌번호, CVC, 비밀번호 |
| **인증 정보** | 로그인 비밀번호, API 키, 시크릿 토큰, SSH 키 |
| **의료 정보** | 진단서, 처방전, 건강검진 결과 |
| **인사 정보** | 다른 직원의 급여, 인사평가, 징계 기록 |
| **영업 비밀** | 미공개 특허 내용, 핵심 기술 코드, M&A 정보 |
| **고객 원본 데이터** | 고객 이름+연락처+주소 등 식별 가능한 조합 |

### 이렇게 바꿔서 입력하세요

```text
❌ 잘못된 예:
"김철수(주민번호: 900101-1234567) 고객의 계약서를 검토해줘"

✅ 올바른 예:
"고객 A의 계약서를 검토해줘. 주요 조건은 다음과 같아:
- 계약 기간: 2년
- 금액: 월 500만원
- 해지 조건: 3개월 전 서면 통보"
```

::: danger 실수했다면 즉시 행동하세요
민감한 정보를 입력했다면, 이 페이지 하단의 **[사고 대응 가이드](#_9-사고-대응-실수로-민감정보를-입력했을-때)**를 바로 확인하세요.
:::

## 5. 사내 보안 정책 체크리스트

Claude Desktop을 업무에 도입하기 전, IT팀 또는 보안팀에 **반드시 확인**해야 할 항목들입니다.

| # | 확인 항목 | 질문 예시 | 확인 여부 |
|---|----------|-----------|:---------:|
| 1 | AI 도구 사용 승인 | "Claude Desktop을 업무에 사용해도 되나요?" | ☐ |
| 2 | 사용 가능 플랜 | "회사에서 어떤 플랜을 사용하고 있나요?" | ☐ |
| 3 | 데이터 분류 기준 | "어떤 등급의 데이터까지 입력 가능한가요?" | ☐ |
| 4 | 허용 업무 범위 | "어떤 업무에 사용할 수 있나요? 제한되는 업무가 있나요?" | ☐ |
| 5 | 파일 접근 범위 | "어떤 폴더/드라이브에 Cowork 접근을 허용해도 되나요?" | ☐ |
| 6 | 개인정보 처리 | "고객 데이터 분석에 사용해도 되나요? 비식별 처리 기준은?" | ☐ |
| 7 | 대화 기록 보관 | "대화 기록은 어떻게 관리하나요? 정기 삭제 정책이 있나요?" | ☐ |
| 8 | 사고 보고 절차 | "민감정보를 실수로 입력했을 때 누구에게 보고하나요?" | ☐ |
| 9 | 교육 요구사항 | "AI 도구 사용 전 별도 보안 교육이 필요한가요?" | ☐ |
| 10 | 계정 관리 | "공용 계정 사용인가요, 개인 계정인가요?" | ☐ |

::: tip 이 표를 프린트해서 사용하세요
위 표를 복사해서 IT팀에 메일로 보내거나, 프린트해서 하나씩 체크하면 편합니다. 한 번만 확인해두면 이후에는 안심하고 사용할 수 있습니다.
:::

## 6. 안전한 사용 가이드라인

### Do — 이렇게 사용하세요

| 실천 항목 | 설명 |
|-----------|------|
| **비식별 처리 후 입력** | 이름 → "고객 A", 연락처 → 삭제, 주민번호 → 삭제 |
| **공개 가능한 정보만 활용** | 이미 공개된 자료, 일반적인 업무 문서 위주로 사용 |
| **작업 폴더 분리** | Cowork 전용 폴더를 만들어 필요한 파일만 복사 |
| **결과물 검증** | Claude가 만든 문서에 민감정보가 섞이지 않았는지 확인 |
| **주기적 대화 정리** | 오래된 대화는 삭제하여 불필요한 데이터 노출 방지 |
| **화면 잠금** | 자리를 비울 때 PC 화면 잠금 (Claude 대화 노출 방지) |
| **업데이트 유지** | Claude Desktop을 항상 최신 버전으로 유지 |

### Don't — 이것은 피하세요

| 금지 항목 | 왜 위험한가 |
|-----------|------------|
| **비밀번호/API 키 입력** | 대화 기록에 남아 유출 위험 |
| **주민번호 등 개인정보 입력** | 개인정보보호법 위반 소지 |
| **기밀 문서 통째로 업로드** | 영업비밀 유출 위험 |
| **다른 사람 계정으로 사용** | 감사 추적 불가, 책임 소재 불명확 |
| **결과물을 검증 없이 외부 전송** | Claude가 생성한 내용에 원치 않는 정보 포함 가능 |
| **공용 PC에서 로그아웃 안 하기** | 다음 사용자가 내 대화와 파일에 접근 가능 |
| **허용되지 않은 폴더 연결** | 민감 파일이 의도치 않게 노출 |

## 7. 감사(Audit) & 로그 관리

### 대화 기록은 어디에 남나요?

| 위치 | 설명 |
|------|------|
| **Claude 서버** | 플랜별 정책에 따라 일정 기간 보관 후 삭제 |
| **내 PC (로컬)** | Claude Desktop 앱 내에 대화 기록이 캐시로 남을 수 있음 |
| **관리자 콘솔** | Team/Enterprise 플랜에서 관리자가 사용 현황을 조회 가능 |

### 대화 기록 삭제 방법

1. **개별 대화 삭제**: 대화 목록에서 해당 대화를 선택 → 우클릭(또는 ⋯ 메뉴) → **삭제**
2. **전체 대화 삭제**: 설정 → 계정 → 대화 기록 관리 → **전체 삭제**
3. **로컬 캐시 삭제**: 설정 → 앱 데이터 초기화

::: warning Enterprise 사용자 참고
Enterprise 플랜에서는 관리자가 감사 로그를 통해 **누가, 언제, 어떤 기능을 사용했는지** 확인할 수 있습니다. 대화 내용 자체를 보는 것은 아니지만, 사용 사실은 기록에 남습니다. 업무 외 사용은 자제하세요.
:::

### 권장 관리 습관

- **매주 금요일**: 불필요한 대화 삭제
- **프로젝트 종료 시**: 관련 대화 일괄 정리
- **민감 업무 완료 후**: 해당 대화 즉시 삭제

## 8. 규제 준수 참고사항

### 개인정보보호법 (한국)

| 원칙 | Claude 사용 시 적용 |
|------|-------------------|
| **수집 최소화** | 업무에 꼭 필요한 데이터만 입력 |
| **목적 외 이용 금지** | 분석 목적으로 받은 데이터를 다른 용도로 Claude에 입력하지 않기 |
| **안전성 확보** | Team/Enterprise 플랜 사용, 비식별 처리 후 입력 |
| **동의 원칙** | 고객 데이터를 AI로 분석한다면, 프라이버시 정책에 반영 필요 |

### GDPR (EU 관련 업무가 있는 경우)

- EU 시민의 개인정보를 Claude에 입력하는 것은 **데이터 이전(Data Transfer)**에 해당할 수 있습니다
- 반드시 법무팀/DPO와 협의하세요
- Enterprise 플랜에서는 DPA(Data Processing Agreement) 체결 가능

### 그 밖의 규제

| 규제/산업 | 주의사항 |
|-----------|---------|
| **금융 (전자금융거래법)** | 고객 금융정보 입력 금지, 내부 통제 절차 확인 |
| **의료 (의료법)** | 환자 정보 입력 금지, HIPAA 대상이면 Enterprise + BAA 필요 |
| **공공기관 (전자정부법)** | AI 도구 사용 가능 여부를 정보보안 담당관에게 확인 |
| **방산/국방** | 보안 등급에 따라 사용 자체가 제한될 수 있음 |

::: tip 확실하지 않다면?
규제 관련 판단은 직접 하지 마세요. **"이 데이터를 Claude에 넣어도 되는지"** IT팀이나 법무팀에 한 번만 물어보면 됩니다. 물어보는 게 사고를 예방하는 가장 확실한 방법입니다.
:::

## 9. 사고 대응: 실수로 민감정보를 입력했을 때

당황하지 마세요. 아래 단계를 **순서대로** 따라하세요.

### 즉시 조치 (5분 이내)

```
1단계: 해당 대화를 즉시 삭제합니다
       → 대화 목록에서 해당 대화 우클릭 → 삭제

2단계: 새 대화를 열어 다시 시작합니다
       (삭제한 대화에서 작업을 이어가지 마세요)

3단계: 어떤 정보가 노출됐는지 메모해둡니다
       → 종류, 건수, 대상자 수 등
```

### 보고 (1시간 이내)

```
4단계: 직속 상사에게 보고합니다

5단계: IT팀/보안팀에 연락합니다
       → "AI 도구에 민감정보를 실수로 입력했습니다"
       → 메모해둔 노출 정보 내용을 전달합니다

6단계: IT팀의 추가 지시를 따릅니다
       → 비밀번호/API 키였다면: 즉시 변경
       → 고객 개인정보였다면: 개인정보 유출 대응 절차 진행
```

### 후속 조치

```
7단계: 재발 방지책을 정합니다
       → 어떤 상황에서 실수했는지 기록
       → 작업 폴더 분리 등 환경 개선

8단계: 필요 시 Anthropic에 데이터 삭제 요청
       → privacy@anthropic.com 으로 연락
       → 계정 정보와 해당 대화 시간대를 알려줍니다
```

::: danger 비밀번호/API 키를 입력한 경우
비밀번호나 API 키를 입력했다면 대화 삭제 **후에 반드시 해당 비밀번호/키를 변경**하세요. 대화를 삭제했더라도 안전을 위해 변경하는 것이 원칙입니다.
:::
