# 온보딩 체크리스트

> Claude Desktop Cowork를 처음 시작하는 분을 위한 단계별 체크리스트입니다. 하나씩 체크하며 따라가 보세요.

::: tip 활용법
이 페이지를 북마크하거나 인쇄해서 책상에 두세요. 항목을 하나씩 완료할 때마다 체크하면 빠짐없이 온보딩할 수 있습니다.
:::

## Day 1: 설치 & 기본 설정

- [ ] Claude Desktop 다운로드 & 설치 ([claude.ai/download](https://claude.ai/download))
- [ ] 계정 로그인 (회사 계정 or 개인 계정)
- [ ] Cowork 모드 활성화
- [ ] 작업 폴더 선택 & 파일 접근 허용
- [ ] 언어 설정 확인 (한국어)
- [ ] 첫 번째 대화 시도 ("안녕, 나는 [이름]이고 [부서]에서 일해")

::: info 첫 대화가 중요합니다
자기소개를 하면 Claude가 맥락을 기억하고, 이후 대화에서 더 적절한 답변을 줍니다. 부서, 역할, 주로 하는 업무를 알려주세요.
:::

## Day 1: 첫 번째 실전 활용

- [ ] 이메일 하나 작성 요청해보기
- [ ] 파일 하나 읽기 & 요약 요청해보기
- [ ] 수정 요청해보기 ("톤을 바꿔줘", "더 짧게")
- [ ] 결과물 저장 or 복사해서 실제 업무에 사용

::: tip 핵심 포인트
처음부터 완벽한 결과를 기대하지 않아도 됩니다. **수정 요청**이 Cowork의 핵심입니다. "좀 더 공손하게", "숫자를 표로 정리해줘"처럼 자유롭게 요청하세요.
:::

## Day 2-3: 익숙해지기

- [ ] Custom Instructions 설정 (팀 규칙, 선호 양식 등)
- [ ] 프로젝트 하나 만들어보기
- [ ] 데이터 파일(엑셀/CSV) 분석 요청해보기
- [ ] 긴 문서 요약 요청해보기
- [ ] 기초 테스트 시나리오 4개 완료

::: info Custom Instructions란?
매번 같은 지시를 반복하지 않도록, 미리 규칙을 저장해두는 기능입니다. 예를 들어 "항상 존댓말로 답변해줘", "보고서는 우리 팀 양식으로 써줘" 같은 설정을 해두면 매 대화에 자동으로 적용됩니다.
:::

## Week 1: 업무에 정착

- [ ] 매일 반복하는 업무 1개를 Claude로 처리
- [ ] 심화 테스트 시나리오 도전
- [ ] 프롬프트 치트시트 즐겨찾기
- [ ] 동료에게 유용한 프롬프트 하나 공유
- [ ] "자주 하는 실수" 페이지 읽기

::: tip 동료와 공유하세요
좋은 프롬프트를 발견하면 팀 채팅방이나 메일로 공유해보세요. 팀 전체의 생산성이 올라갑니다.
:::

## Week 2+: 파워 유저

- [ ] MCP 연동 탐색 (외부 도구와 Claude 연결)
- [ ] 팀 프로젝트 활용 (팀원들과 같은 프로젝트에서 협업)
- [ ] 나만의 프롬프트 템플릿 3개 이상 만들기
- [ ] 월간 업무에 Claude 활용 루틴 정착
