# 프레젠테이션 활용법

> 발표 준비에 가장 시간이 많이 드는 부분을 Claude가 도와줍니다.

## PT 준비 단계별 활용

### 1단계: 구조 잡기

```text
[주제]에 대한 프레젠테이션 구조를 잡아줘.
대상: [누구에게 발표하는지]
시간: [발표 시간]
목적: [설득/보고/교육/제안]
핵심 메시지: [하나의 핵심 메시지]
슬라이드 수와 각 슬라이드의 제목 + 핵심 포인트를 정리해줘.
```

### 2단계: 슬라이드 내용 작성

```text
[N]번 슬라이드의 내용을 작성해줘.
제목: [슬라이드 제목]
이 슬라이드에서 전달할 핵심: [핵심 포인트]
포함할 것: [데이터/차트/인용구/사례]
글자 수: 슬라이드당 최대 30단어 (발표자 노트는 별도)
발표자 노트도 같이 작성해줘.
```

### 3단계: 데이터 시각화 제안

```text
이 데이터를 발표용으로 시각화하려면 어떤 차트가 좋을까?
[데이터 설명]
각 차트 유형별로 장단점을 알려주고, 가장 추천하는 것과 이유를 설명해줘.
```

### 4단계: 스토리텔링 강화

```text
이 프레젠테이션에 스토리텔링 요소를 추가해줘.
현재 구조: [구조]
청중이 공감할 수 있는 도입부 후보 3가지를 제안해줘.
- 질문으로 시작
- 에피소드/사례로 시작
- 충격적인 데이터로 시작
```

### 5단계: Q&A 준비

```text
이 발표 후 예상되는 질문 10개와 각각의 모범 답변을 준비해줘.
발표 주제: [주제]
청중: [대상]
까다로운 질문도 포함해줘.
```

### 6단계: 리허설 & 다듬기

```text
아래 발표 스크립트를 리뷰해줘:
[스크립트]

다음을 점검해줘:
- 시간 배분은 적절한지 (목표: [X]분)
- 논리 흐름이 자연스러운지
- 중복되는 내용은 없는지
- 청중 입장에서 지루한 구간은 없는지
- 개선 제안을 구체적으로
```

## 상황별 PT 템플릿

### 경영진 보고 (15분)

- **구조:** 결론 → 근거 → 요청사항
- **팁:** 첫 슬라이드에 결론, 데이터는 부록으로

### 팀 교육 (30분)

- **구조:** 도입 → 개념 → 실습 → 정리
- **팁:** 실습/퀴즈 포함, 일방적 전달 피하기

### 고객 제안 (45분)

- **구조:** 고객 현황 → 문제점 → 솔루션 → 효과 → 일정/비용
- **팁:** 고객의 언어로, ROI 수치 강조

### 사내 공유 (10분)

- **구조:** 배경 → 내용 → 시사점 → 액션
- **팁:** 핵심만, 슬라이드 10장 이내

## PT에서 절대 하면 안 되는 것 5가지

| 실수 | 왜 문제인지 | 대안 |
|------|------------|------|
| 슬라이드에 글자를 빽빽하게 채우기 | 청중은 읽기와 듣기를 동시에 못 합니다 | 슬라이드당 30단어 이내, 나머지는 발표자 노트에 |
| 데이터만 나열하고 해석 없이 넘기기 | 숫자 자체는 의미가 아닙니다 | "이 숫자가 의미하는 것은..." 한 문장 추가 |
| 청중 분석 없이 준비하기 | 같은 내용도 대상에 따라 전달 방식이 달라야 합니다 | 발표 전 "청중이 가장 궁금해할 것"을 3가지 적어보기 |
| 리허설 없이 본번 발표하기 | 시간 초과, 흐름 끊김, 당황 등 예방 불가 | 최소 2회 소리 내어 연습, 타이머 필수 |
| Q&A를 준비하지 않기 | 예상 못한 질문에 당황하면 발표 전체 인상이 흔들립니다 | Claude로 예상 질문 10개와 모범 답변 미리 준비 |

::: tip 연습이 최고의 발표를 만듭니다
아무리 좋은 슬라이드와 스크립트가 있어도 연습 없이는 좋은 발표가 나올 수 없습니다. Claude가 구조와 내용을 잡아주면, 여러분은 연습에 집중하세요. 반복 리허설만이 자신감 있는 발표를 만듭니다.
:::
