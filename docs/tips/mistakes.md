# 자주 하는 실수 & 해결법

> 처음 사용할 때 흔히 겪는 문제들과 해결 방법입니다.

## 1. 너무 짧게 부탁한다

- ❌ "보고서 써줘"
- ✅ "3월 영업 실적 보고서를 써줘. A4 2장, 표 포함, 전월 대비 비교, 팀장님께 보고할 거야"

**해결법:** 동료에게 부탁한다고 생각하세요. 동료에게도 그렇게 짧게 말하면 "뭘 쓰라고?" 라고 되물을 거예요.

## 2. 한 번에 완벽한 결과를 기대한다

Claude는 **초안 작성기**라고 생각하세요. 초안을 받고 → 피드백을 주고 → 수정을 받는 과정이 핵심입니다. 보통 2~3번 주고받으면 원하는 수준이 됩니다.

유용한 피드백:
- "전체적으로 좋은데, 3번 항목을 더 구체적으로"
- "톤이 너무 딱딱해. 좀 더 부드럽게"
- "길이가 너무 길어. 반으로 줄여줘"

## 3. 새 대화에서 이전 맥락을 기대한다

새 대화를 시작하고 "아까 그거 이어서 해줘"라고 하면 Claude가 모릅니다.

- 같은 작업을 이어하려면 **같은 대화 세션**을 유지하세요.
- 새 대화를 열어야 한다면, 필요한 맥락을 다시 알려주세요.

## 4. 민감한 정보를 그대로 입력한다

::: danger 주의
주민등록번호, 비밀번호, 개인 카드번호 등을 입력하지 마세요. 필요한 경우 "XXX-XX-XXXXX" 처럼 마스킹 처리하세요.
:::

## 5. Claude의 답변을 검증 없이 사용한다

- **팩트 체크:** 수치, 날짜, 이름 등은 반드시 확인하세요
- **톤 체크:** 보내기 전에 한 번 읽어보세요
- Claude에게 물어볼 수도 있어요: "이 내용에서 사실 확인이 필요한 부분을 알려줘"

## 6. 파일 이름을 정확히 안 알려준다

- ❌ "그 엑셀 파일 열어봐"
- ✅ "매출_2026Q1.xlsx" 또는 "내 문서 폴더에서 '매출'이 들어간 엑셀 파일 찾아줘"

## 7. 너무 많은 걸 한꺼번에 부탁한다

한 번에 3~5가지까지가 적당합니다. 큰 작업은 단계별로 나눠서 요청하세요.

- "일단 목차부터 잡아줘" → (확인) → "이제 1장을 써줘" → (확인) → ...

## 8. 할루시네이션(거짓 정보)을 못 알아챈다

Claude는 가끔 **그럴듯하지만 틀린 정보**를 자신 있게 말합니다. 이를 "할루시네이션"이라고 합니다.

**특히 주의할 때:**
- 구체적인 수치나 통계를 인용할 때
- 법률, 규정, 제도를 설명할 때
- 특정 인물의 발언을 인용할 때
- "~에 따르면" 형태의 출처 언급 시

**검증 방법:**
- 수치는 원본 데이터와 대조
- 중요한 사실은 공식 자료로 재확인
- Claude에게 물어보세요: "방금 말한 내용의 출처가 있어? 확실해?"
- 확인이 어려운 내용은 "Claude 추천 의견" 등으로 표시하고 사용

::: danger 중요
Claude는 "모른다"고 말하기보다 그럴듯한 답을 만들어낼 수 있습니다. **사실 확인은 항상 사용자의 몫**입니다.
:::

::: tip 기억하세요
Claude는 완벽한 도구가 아니라, **매우 유능한 동료**입니다. 명확하게 설명하고, 결과를 확인하고, 필요하면 수정을 요청하세요.
:::
