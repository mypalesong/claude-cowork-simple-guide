# 문제 해결 가이드

> 문제가 생겼을 때 당황하지 마세요. 대부분의 문제는 간단한 조치로 해결됩니다. 증상별로 원인과 해결 방법을 정리했습니다.

::: tip 먼저 시도해 보세요
어떤 문제든 아래 3가지를 먼저 시도하면 절반 이상 해결됩니다.
1. Claude Desktop을 완전히 종료하고 다시 실행
2. 인터넷 연결 상태 확인
3. 앱을 최신 버전으로 업데이트
:::

## 설치 & 접속 문제

### **Claude Desktop이 설치되지 않아요**

| 항목 | 내용 |
|------|------|
| 증상 | 설치 파일 실행 시 오류 발생 또는 설치가 진행되지 않음 |
| 원인 | 시스템 요구사항 미충족, 보안 소프트웨어 차단, 디스크 용량 부족 |

**해결 방법:**
1. 시스템 요구사항을 확인하세요 (Windows 10 이상, macOS 12 이상)
2. 디스크 여유 공간이 최소 500MB 이상인지 확인하세요
3. 회사 보안 프로그램(백신, DLP 등)이 설치를 차단하고 있다면, IT 부서에 설치 허용을 요청하세요
4. 관리자 권한으로 설치 파일을 실행하세요 (Windows: 우클릭 → 관리자 권한으로 실행)
5. 기존 설치 파일을 삭제한 후 [claude.ai/download](https://claude.ai/download)에서 최신 버전을 다시 다운로드하세요

::: warning 회사 컴퓨터의 경우
회사 보안 정책으로 프로그램 설치가 제한된 환경에서는 반드시 IT 부서의 승인을 먼저 받으세요.
:::

**그래도 안 되면:** IT 부서에 설치 로그 파일을 첨부하여 지원을 요청하세요.

---

### **로그인이 안 돼요**

| 항목 | 내용 |
|------|------|
| 증상 | 로그인 버튼을 눌러도 반응이 없거나, 인증 페이지가 열리지 않음 |
| 원인 | 브라우저 문제, 네트워크 차단, 계정 문제 |

**해결 방법:**
1. 기본 브라우저가 정상 작동하는지 확인하세요
2. [claude.ai](https://claude.ai)에 웹 브라우저로 직접 접속하여 로그인이 되는지 테스트하세요
3. 회사 방화벽이 `*.anthropic.com`, `*.claude.ai` 도메인을 차단하고 있지 않은지 확인하세요
4. VPN을 사용 중이라면 일시적으로 해제 후 다시 시도하세요
5. Claude Desktop 앱의 캐시를 삭제한 후 재시작하세요

**그래도 안 되면:** [support.anthropic.com](https://support.anthropic.com)에서 계정 관련 문의를 접수하세요.

---

### **Cowork 탭이 보이지 않아요**

| 항목 | 내용 |
|------|------|
| 증상 | Claude Desktop에 Cowork 관련 메뉴나 탭이 표시되지 않음 |
| 원인 | 무료 플랜 사용 중, 앱 버전이 오래됨, 기능이 비활성화 상태 |

**해결 방법:**
1. 현재 사용 중인 플랜을 확인하세요 — Cowork 기능은 **Pro 이상**에서 사용 가능합니다
2. Claude Desktop을 최신 버전으로 업데이트하세요
3. 설정(Settings) → 기능(Features)에서 Cowork 관련 토글이 켜져 있는지 확인하세요
4. 앱을 완전히 종료 후 재시작하세요

**그래도 안 되면:** 플랜 정보와 앱 버전을 확인한 후 고객지원에 문의하세요.

### **"연결할 수 없습니다" 오류**

| 항목 | 내용 |
|------|------|
| 증상 | 대화 시도 시 "서버에 연결할 수 없습니다" 또는 네트워크 오류 메시지 표시 |
| 원인 | 인터넷 연결 끊김, 서버 점검 중, 방화벽 차단 |

**해결 방법:**
1. 인터넷 연결 상태를 확인하세요 — 웹 브라우저에서 다른 사이트가 열리는지 테스트
2. [status.anthropic.com](https://status.anthropic.com)에서 서버 상태를 확인하세요
3. Wi-Fi를 끄고 다시 연결하거나, 유선 연결로 전환해 보세요
4. 프록시 또는 VPN 설정을 확인하세요
5. 방화벽에서 Claude Desktop의 네트워크 접근을 허용하세요

::: tip 서버 점검인 경우
Anthropic은 간헐적으로 서버 점검을 진행합니다. [status.anthropic.com](https://status.anthropic.com)에서 점검 일정을 확인하고, 잠시 후 다시 시도하세요.
:::

**그래도 안 되면:** 네트워크 환경 정보(프록시 사용 여부, VPN 등)를 포함하여 IT 부서 또는 고객지원에 문의하세요.

## 파일 관련 문제

### **파일을 읽지 못해요**

| 항목 | 내용 |
|------|------|
| 증상 | 파일 이름을 알려줬는데 "파일을 찾을 수 없습니다"라고 응답 |
| 원인 | 파일 경로 오류, 파일 이름 오타, 지원하지 않는 파일 형식 |

**해결 방법:**
1. 파일 이름과 확장자를 정확히 입력했는지 확인하세요
2. 파일의 전체 경로를 알려주세요 (예: `C:\Users\홍길동\Documents\보고서.xlsx`)
3. 파일 탐색기에서 파일이 실제로 존재하는지 확인하세요
4. "내 문서 폴더에서 '매출'이 들어간 파일을 찾아줘"라고 요청해 보세요
5. 파일이 다른 프로그램에서 열려 있다면 닫은 후 다시 시도하세요

**그래도 안 되면:** 파일 경로를 복사하여 대화창에 직접 붙여넣기 해 보세요.

---

### **"접근 권한이 없습니다" 오류**

| 항목 | 내용 |
|------|------|
| 증상 | 파일에 접근하려 하면 권한 오류 메시지 표시 |
| 원인 | 파일/폴더 권한 설정, Claude Desktop에 파일 접근 권한 미부여 |

**해결 방법:**
1. Claude Desktop의 설정에서 **파일 접근 허용 폴더** 목록을 확인하세요
2. 해당 파일이 포함된 폴더를 허용 목록에 추가하세요
3. 파일이 읽기 전용으로 설정되어 있다면, 파일 속성에서 읽기 전용을 해제하세요
4. 네트워크 드라이브의 파일인 경우, 로컬에 복사한 후 시도하세요

::: danger 보안 주의
회사 보안 폴더나 암호화된 드라이브의 파일은 의도적으로 접근이 제한될 수 있습니다. 무분별하게 권한을 변경하지 마세요.
:::

**그래도 안 되면:** 해당 파일을 허용된 폴더(예: 바탕화면, 내 문서)로 복사한 후 다시 시도하세요.

---

### **파일을 저장하지 못해요**

| 항목 | 내용 |
|------|------|
| 증상 | Claude가 파일을 생성하거나 수정한 후 저장에 실패 |
| 원인 | 쓰기 권한 없음, 디스크 용량 부족, 파일이 다른 프로그램에서 사용 중 |

**해결 방법:**
1. 저장할 폴더에 쓰기 권한이 있는지 확인하세요
2. 디스크 여유 공간을 확인하세요
3. 같은 이름의 파일이 다른 프로그램(엑셀, 워드 등)에서 열려 있다면 닫으세요
4. 다른 폴더(예: 바탕화면)에 저장해 보세요
5. 파일 이름에 특수문자(`\`, `/`, `*`, `?` 등)가 포함되어 있지 않은지 확인하세요

**그래도 안 되면:** Claude에게 내용을 대화창에 출력하도록 요청한 후 직접 복사하여 저장하세요.

### **엑셀 파일을 열 수 없어요**

| 항목 | 내용 |
|------|------|
| 증상 | `.xlsx` 파일을 읽으라고 했는데 내용을 읽지 못하거나 깨진 데이터를 표시 |
| 원인 | 파일이 암호로 보호됨, 매크로/VBA 포함, 파일 손상 |

**해결 방법:**
1. 파일에 **비밀번호가 설정**되어 있다면, 엑셀에서 비밀번호를 해제한 후 다시 저장하세요
2. `.xlsm` (매크로 포함) 파일은 `.xlsx`로 다시 저장한 후 시도하세요
3. 파일 크기가 너무 크다면(50MB 이상), 필요한 시트만 별도 파일로 저장하세요
4. `.csv`로 변환하여 시도하면 더 안정적으로 읽힙니다

::: tip CSV 변환 방법
엑셀에서 파일 → 다른 이름으로 저장 → 파일 형식을 "CSV (쉼표로 분리)"로 선택하세요.
:::

**그래도 안 되면:** 필요한 데이터 범위를 선택하여 새 엑셀 파일에 붙여넣기 후 시도하세요.

---

### **한글(.hwp) 파일이 안 열려요**

| 항목 | 내용 |
|------|------|
| 증상 | `.hwp` 또는 `.hwpx` 파일을 읽으라고 했는데 내용을 표시하지 못함 |
| 원인 | HWP는 한글과컴퓨터 고유 포맷으로, 직접 지원이 제한적 |

**해결 방법:**
1. 한글 프로그램에서 해당 파일을 열고 **다른 이름으로 저장** → `.docx` 또는 `.pdf`로 변환하세요
2. 변환된 파일을 Claude에게 전달하세요
3. 간단한 문서라면 한글에서 텍스트를 복사하여 대화창에 붙여넣기 하세요
4. `.hwpx` 형식(한글 2014 이상)이 `.hwp`보다 호환성이 좋으니, 가능하면 `.hwpx`로 저장하세요

::: warning HWP 파일 변환
공공기관 문서 등 서식이 중요한 경우, 변환 시 레이아웃이 달라질 수 있습니다. 원본은 반드시 보관하세요.
:::

**그래도 안 되면:** 한글 뷰어에서 텍스트를 복사하여 `.txt` 파일로 저장한 후 전달하세요.

## 대화 & 응답 문제

### **대화가 갑자기 끊겨요**

| 항목 | 내용 |
|------|------|
| 증상 | 응답 도중 멈추거나, 대화가 중간에 끊기고 더 이상 진행되지 않음 |
| 원인 | 컨텍스트 윈도우(대화 길이 한도) 초과, 네트워크 일시 끊김, 서버 과부하 |

**해결 방법:**
1. "계속해줘" 또는 "이어서 작성해줘"라고 입력하세요
2. 네트워크 연결을 확인하세요
3. 대화가 너무 길어졌다면(수십 회 이상), **새 대화를 시작**하고 핵심 맥락만 다시 전달하세요
4. 큰 작업은 단계별로 나누어 요청하세요 (예: "일단 1장만 써줘")

**그래도 안 되면:** 새 대화를 열고 이전 대화의 핵심 내용을 요약하여 전달한 후 이어서 작업하세요.

---

### **응답이 너무 느려요**

| 항목 | 내용 |
|------|------|
| 증상 | 질문 후 응답이 나오기까지 수십 초 이상 걸리거나, 글자가 매우 천천히 표시됨 |
| 원인 | 서버 사용량 폭주, 대용량 파일 처리 중, 네트워크 속도 저하 |

**해결 방법:**
1. [status.anthropic.com](https://status.anthropic.com)에서 서버 상태를 확인하세요
2. 요청하는 작업의 범위를 줄여보세요 (파일 크기, 요청 분량 등)
3. 피크 타임(한국 시간 오전 9~11시)을 피해 시도해 보세요
4. Wi-Fi 대신 유선 연결을 사용하거나, 다른 네트워크로 전환하세요
5. 불필요한 백그라운드 프로그램을 종료하세요

**그래도 안 되면:** 잠시 후 다시 시도하세요. 지속적으로 느리다면 고객지원에 문의하세요.

### **이전 맥락을 기억 못 해요**

| 항목 | 내용 |
|------|------|
| 증상 | "아까 그거"라고 했는데 못 알아듣거나, 이전에 알려준 내용을 모름 |
| 원인 | 새 대화 세션에서는 이전 대화 내용을 참조하지 못함, 대화가 너무 길어서 초기 내용이 잘림 |

**해결 방법:**
1. **같은 대화 세션** 안에서 이어서 작업하세요 — 새 대화를 시작하면 기억이 초기화됩니다
2. 새 대화에서 이어할 때는 필요한 맥락을 다시 알려주세요
3. 프로젝트 설정 파일에 자주 쓰는 맥락 정보를 저장해 두세요
4. 긴 대화에서 중요한 내용은 중간중간 요약을 요청하세요

::: tip 맥락 전달 꿀팁
새 대화를 시작할 때 이렇게 말해보세요: "이전 대화에서 3월 영업 보고서를 작성 중이었어. 목차는 이거고, 1장까지 완성했어. 이어서 2장을 작성해줘."
:::

**그래도 안 되면:** 중요한 대화 내용은 파일로 저장해두고, 새 대화에서 해당 파일을 참조하도록 요청하세요.

### **한국어 응답이 어색해요**

| 항목 | 내용 |
|------|------|
| 증상 | 번역 투 표현, 부자연스러운 존댓말, 어색한 어미 사용 |
| 원인 | 구체적인 톤/스타일 지시 없이 요청, 영어 기반 학습 데이터의 영향 |

**해결 방법:**
1. 원하는 톤을 명시하세요: "격식체로", "~입니다 체로", "친근한 반말로"
2. 대상을 알려주세요: "팀장님께 보고할 문서", "신입사원 교육용 자료"
3. 예시를 보여주세요: "이런 느낌으로 써줘: [원하는 문장 예시]"
4. 어색한 부분을 짚어주세요: "3번째 문단이 번역 투야. 자연스러운 한국어로 고쳐줘"

**그래도 안 되면:** 초안을 받은 후 "이 문서 전체를 자연스러운 비즈니스 한국어로 다듬어줘"라고 요청하세요.

### **엉뚱한 답을 해요 (할루시네이션)**

| 항목 | 내용 |
|------|------|
| 증상 | 그럴듯하지만 사실과 다른 정보를 자신 있게 말함, 존재하지 않는 출처 인용 |
| 원인 | AI 모델의 본질적 한계, 학습 데이터에 없는 최신 정보 요청, 모호한 질문 |

**해결 방법:**
1. 구체적인 수치, 날짜, 법률 등은 반드시 **별도로 사실 확인**하세요
2. "확실하지 않은 부분은 알려줘"라고 미리 요청하세요
3. 출처가 필요한 경우 "출처를 함께 알려줘. 확실하지 않으면 확인이 필요하다고 표시해줘"라고 말하세요
4. 질문을 더 구체적으로 해 주세요 — 모호할수록 할루시네이션 가능성이 높아집니다
5. 최신 정보가 필요하면 관련 자료를 파일로 전달하고 "이 자료를 기반으로 답해줘"라고 요청하세요

::: danger 반드시 검증하세요
특히 다음 내용은 Claude의 답변을 그대로 신뢰하지 마세요:
- 법률, 규정, 제도 관련 내용
- 특정 수치, 통계 데이터
- 인물의 발언 인용
- "~에 따르면" 형태의 출처 언급
:::

**그래도 안 되면:** 해당 주제에 대해 신뢰할 수 있는 자료를 직접 확인한 후, 그 자료를 Claude에게 전달하여 작업하세요.

## 기능 관련 문제

### **MCP 연동이 안 돼요**

| 항목 | 내용 |
|------|------|
| 증상 | MCP 서버를 설정했는데 연결되지 않거나, 도구가 표시되지 않음 |
| 원인 | 설정 파일 오류, MCP 서버 미실행, 경로 설정 오류 |

**해결 방법:**
1. `claude_desktop_config.json` 파일의 문법(JSON 형식)이 올바른지 확인하세요
2. MCP 서버 경로가 정확한지 확인하세요 — 오타가 가장 흔한 원인입니다
3. 필요한 런타임(Node.js, Python 등)이 설치되어 있는지 확인하세요
4. Claude Desktop을 완전히 종료 후 재시작하세요 — 설정 파일은 시작 시에만 읽습니다
5. MCP 서버를 터미널에서 직접 실행하여 오류 메시지를 확인하세요

::: warning JSON 형식 주의
쉼표(`,`) 누락, 중괄호(`{}`) 짝 불일치가 가장 흔한 설정 오류입니다. JSON 검증 도구를 사용하여 문법을 확인하세요.
:::

**그래도 안 되면:** MCP 서버의 로그 파일을 확인하거나, 해당 MCP 도구의 공식 문서를 참고하세요.

---

### **플러그인이 작동하지 않아요**

| 항목 | 내용 |
|------|------|
| 증상 | 설치한 플러그인이 목록에 표시되지 않거나, 사용 시 오류 발생 |
| 원인 | 플러그인 버전 호환성 문제, 설치 오류, 앱 버전이 오래됨 |

**해결 방법:**
1. Claude Desktop을 최신 버전으로 업데이트하세요
2. 플러그인을 삭제하고 다시 설치하세요
3. 다른 플러그인과 충돌이 없는지 확인하세요 — 하나씩 비활성화하여 테스트
4. 플러그인의 필수 의존성(다른 프로그램, 라이브러리 등)이 설치되어 있는지 확인하세요

**그래도 안 되면:** 해당 플러그인의 GitHub 이슈 페이지에서 비슷한 문제가 보고되었는지 확인하세요.

### **팀 공유가 안 돼요**

| 항목 | 내용 |
|------|------|
| 증상 | 팀원에게 프로젝트나 대화를 공유하려는데 공유 옵션이 없거나 실패 |
| 원인 | 개인(Pro) 플랜 사용 중, 팀 설정 미완료, 팀원 초대 미완료 |

**해결 방법:**
1. 현재 플랜이 **Team 이상**인지 확인하세요 — 개인 Pro 플랜에서는 팀 공유가 제한됩니다
2. 관리자 콘솔에서 팀원이 정상적으로 초대되었는지 확인하세요
3. 공유 대상 팀원이 초대를 수락하고 계정 설정을 완료했는지 확인하세요
4. 공유하려는 항목의 접근 권한 설정을 확인하세요

**그래도 안 되면:** 팀 관리자에게 팀 설정 상태 확인을 요청하세요.

---

### **예약 작업이 실행되지 않아요**

| 항목 | 내용 |
|------|------|
| 증상 | 예약한 자동 작업이 설정한 시간에 실행되지 않음 |
| 원인 | 컴퓨터가 꺼져 있거나 절전 모드, Claude Desktop 미실행, 예약 설정 오류 |

**해결 방법:**
1. 예약 시간에 컴퓨터가 **켜져 있고 절전 모드가 아닌지** 확인하세요
2. Claude Desktop이 실행 중인 상태여야 합니다 — 시작 프로그램에 등록하세요
3. 예약 설정에서 시간대(Time Zone)가 **한국 시간(KST, UTC+9)**으로 되어 있는지 확인하세요
4. 예약 작업을 삭제하고 다시 생성해 보세요

**그래도 안 되면:** 수동으로 해당 작업을 실행하여 작업 자체에 문제가 없는지 먼저 확인하세요.

## 계정 & 요금 문제

### **사용량 한도에 도달했어요**

| 항목 | 내용 |
|------|------|
| 증상 | "사용량 한도에 도달했습니다" 메시지가 표시되고 더 이상 응답하지 않음 |
| 원인 | 현재 플랜의 일일/월별 사용량 제한 초과 |

**해결 방법:**
1. 현재 사용량을 설정 → 사용량(Usage)에서 확인하세요
2. 한도 초기화 시점을 확인하세요 (보통 매일 또는 매월 갱신)
3. 급한 작업이 있다면 플랜 업그레이드를 고려하세요
4. 불필요하게 긴 대화나 반복 요청을 줄이면 사용량을 절약할 수 있습니다

| 플랜 | 사용량 한도 | 초기화 주기 |
|------|-----------|-----------|
| 무료 | 매우 제한적 | 매일 |
| Pro | 넉넉함 | 매일 |
| Team | 팀원별 할당 | 매월 |
| Enterprise | 맞춤 설정 | 계약에 따름 |

**그래도 안 되면:** 플랜 업그레이드를 진행하거나, 한도 초기화 시점까지 기다리세요.

---

### **플랜 업그레이드 방법**

| 항목 | 내용 |
|------|------|
| 증상 | 더 많은 기능이나 사용량이 필요하여 플랜을 변경하고 싶음 |

**해결 방법:**
1. [claude.ai](https://claude.ai)에 로그인하세요
2. 설정(Settings) → 요금제(Subscription)로 이동하세요
3. 원하는 플랜을 선택하고 결제 정보를 입력하세요
4. 업그레이드는 즉시 적용되며, 기존 결제 주기에 비례하여 청구됩니다

::: tip 회사에서 도입하는 경우
Team/Enterprise 플랜은 관리자가 중앙에서 관리합니다. 개인이 별도 결제하지 말고, 회사 관리자에게 요청하세요.
:::

**그래도 안 되면:** 결제 관련 문제는 [support.anthropic.com](https://support.anthropic.com)에 문의하세요.

---

### **팀 멤버 추가 방법**

| 항목 | 내용 |
|------|------|
| 증상 | 새로운 팀원을 Cowork에 추가하고 싶음 |

**해결 방법:**
1. **관리자 계정**으로 [claude.ai](https://claude.ai)에 로그인하세요
2. 관리자 콘솔(Admin Console) → 팀원 관리(Members)로 이동하세요
3. "초대하기(Invite)" 버튼을 클릭하세요
4. 팀원의 **회사 이메일 주소**를 입력하고 역할(관리자/멤버)을 설정하세요
5. 초대된 팀원이 이메일의 링크를 클릭하여 가입을 완료하면 사용 가능합니다

::: warning 권한 관리
관리자(Admin) 역할은 팀 설정 변경, 멤버 추가/삭제가 가능합니다. 필요한 인원에게만 관리자 권한을 부여하세요.
:::

**그래도 안 되면:** Team/Enterprise 플랜의 좌석(seat) 수가 부족할 수 있습니다. 관리자 콘솔에서 잔여 좌석을 확인하거나 추가 구매하세요.
