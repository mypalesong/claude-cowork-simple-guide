# 기초 테스트 시나리오

> 처음 접하는 분도 바로 따라할 수 있는 테스트입니다. 순서대로 진행해보세요!

::: info 테스트 방법
각 시나리오를 순서대로 Claude에게 입력하고, 결과를 확인하세요. **확인 포인트**와 비교해보면 Cowork가 어떤 수준인지 체감할 수 있습니다.
:::

## Test 1: 기본 대화 능력

**목표:** Claude가 맥락을 이해하고 이어서 대화하는지 확인

**Step 1 — 자기소개:**
```text
나는 마케팅팀 대리야. 우리 팀은 5명이고, B2B SaaS 제품을 담당하고 있어.
```

**Step 2 — 맥락 기억 테스트:**
```text
우리 제품의 경쟁사 분석 보고서 목차를 잡아줘.
```
> ✅ **확인:** "B2B SaaS"라는 맥락을 반영했는가? 마케팅 관점의 분석인가?

**Step 3 — 수정 요청:**
```text
2번 섹션을 좀 더 구체적으로 나눠줘. 그리고 가격 비교 섹션도 추가해.
```
> ✅ **확인:** "2번 섹션"이 무엇인지 정확히 이해했는가?

## Test 2: 파일 읽기 & 요약

**목표:** Claude가 내 파일을 제대로 읽고 분석하는지 확인

아무 문서 파일(.docx, .txt, .pdf)을 작업 폴더에 넣어두고:

```text
"[파일명]" 파일을 읽고 다음을 알려줘:
1. 이 문서가 무엇에 대한 건지 한 줄 요약
2. 핵심 내용 3가지
3. 내가 꼭 알아야 할 것 1가지
```
> ✅ **확인:** 파일을 정확히 읽었는가? 요약이 핵심을 잘 짚었는가?

## Test 3: 문서 작성 품질

**목표:** 실제 업무에서 바로 쓸 수 있는 품질인지 확인

```text
아래 상황으로 공식 이메일을 작성해줘:

상황: 우리 회사가 주최하는 세미나(4월 15일, 서울 코엑스)에
파트너사 임원들을 초대하는 이메일.

조건:
- 세미나 주제: "2026년 디지털 전환 전략"
- 기조연설: 우리 회사 CEO 김영수
- 참석 혜택: 네트워킹 디너, 백서 제공
- RSVP 기한: 4월 5일까지
톤: 격식 있되 딱딱하지 않게, 참석하고 싶게 만드는 느낌으로
```

> ✅ **확인:** 이메일 제목이 매력적인가? 모든 정보가 빠짐없는가? 톤이 적절한가? 실제로 보내도 될 수준인가?

## Test 4: 표 & 데이터 정리

**목표:** 비정형 데이터를 깔끔하게 정리하는지 확인

```text
아래 메모를 깔끔한 표로 정리해줘:

1월 매출 3200만 서울지점
2월은 서울 2800 부산 1500
3월 서울지점 3500만원 부산점 1800만원 대전 900
1월 부산 1200만 대전 800만
2월 대전 750만원

이걸로:
1. 지점별 x 월별 매출표 (단위: 만원)
2. 합계 행과 열 추가
3. 전월 대비 증감률도 표시
```
> ✅ **확인:** 흩어진 데이터를 정확히 파악했는가? 표가 깔끔한가? 계산이 맞는가?
